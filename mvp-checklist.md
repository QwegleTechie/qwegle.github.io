# MVP Build Checklist (Quick Reference)

## Product
- One clear problem
- One primary user flow
- One success metric

## Engineering
- Clean separation of concerns
- No premature abstractions
- Code readable by a new developer

## UX
- No dead ends
- Clear next steps
- Obvious error recovery

## Launch
- Logging enabled
- Monitoring in place
- Feedback channel open

This checklist favors long term stability over short-term hacks.
